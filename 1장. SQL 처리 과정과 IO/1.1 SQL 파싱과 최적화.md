#### 1.1.1 구조적, 집합적, 선언적 질의 언어
SQL(Structured Query Language)
- 구조적
- 집합적
- 선언적
- 결과집합을 만드는 과정은 절차적(프로시저)

사용자 ->(sql)-> 옵티마이저 ->(실행계획)-> 프로시저

SQL 최적화란
- DBMS 내부에서 프로시저를 작성하고 컴파일해서 실행 가능한 상태로 만드는 전 과정

<br>

#### 1.1.2 SQL 최적화

 SQL 최적화 과정
 1) SQL 파싱
    - 파싱 트리 생성, 문법 및 의미상 오류 체크
 2) SQL 최적화
    - 옵티마이저가 담당
    - 미리 수집한 시스템 및 오브젝트 통계정보를 바탕으로 실행경로 선택 
 3) 로우 소스 생성
    - 로우 소스 생성기가 담당
    - 실행경로를 실제 실행 가능한 코드 또는 프로시저 형태로 포맷팅

<br>

#### 1.1.3 SQL 옵티마이저

옵티마이저 최적화 단계
1) 사용자로부터 전달받은 쿼리를 수행할 실행계획 후보군 찾기
2) 데이터 딕셔너리에 수집해 둔 정보로 각 실행계획의 예상비용 산정
3) 최저비용을 나타내는 실행계획 선택
