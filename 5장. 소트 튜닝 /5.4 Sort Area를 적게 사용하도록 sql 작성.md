### 5.4 Sort Area를 적게 사용하도록 sql 작성

- 소트 연산이 불가피하다면, 메모리 내에서 처리 완료하도록 할 것
- Sort Area를 적게 사용할 방법을 찾아야 함

<br>

- Top N 소트 알고리즘
  - 전체 데이터셋을 정렬하지 않고 상위 N개(가장 크거나 작은) 데이터만 효율적으로 추출하는 방식
  - 실행계획 : SORT ORDER BY STOPKEY
  - 소트 연산 횟수와 Sort Area 사용량 최소화
  -  sql에 rownum 조건절을 삭제하면 'Top N 소트 알고리즘'이 작동하지 않음

<br>

- 분석함수에서의 Top N 소트
  - rank, row_number 함수는 max 함수보다 소트 부하가 적음
  - Top N 소트 알고리즘이 작동하기 때문


<br>
