### 6.1 기본 DML 튜닝

- 인덱스와 DML 성능
  - 인덱스를 하나라도 줄이면, TPS 성능 향상
  - UPDATE 시 인덱스 삭제 후 삽입, 총 2번 작업 발생 

<BR>

- 무결성 제약과 DML 성능

<BR>

- REDO 로깅
- UNDO 로깅

<BR>

- LOCK


<BR>

- 커밋
  - 커밋과 DML은 별개로 실행하지만, DML을 끝내려면 커밋까지 완료해야 하므로 서로 밀접함
  - 커밋 과정
    - DML 실행, REDO 로그버퍼에 변경 기록
    - 데이터 변경
    - 커밋
    - 로그버퍼 -> 파일 저장
    - 변경된 블록 -> 데이터 파일 저장
  - Write Ahead Logging : 오라클은 데이터를 변경하기 전에 항상 로그부터 기록함
  - Log Force at Commit : '적어도 커밋시점에는' Redo 로그버퍼 내용을 로그파일에 기록한다


<BR>

- 데이터베이스 Call
  - Parse Call : SQL 파싱과 최적화를 수행하는 단계
  - Execute Call : SQL 실행 단계
  - Fetch Call : 데이터를 읽어서 사용자에게 결과집합을 전송하는 과정 (SELECT문 ONLY)
- Call 발생 위치에 따른 분류
  - User Call : 네트워크를 경유해 dbms 외부로부터 반입되는 Call / 성능에 미치는 영향이 매우 큼
  - Recursive Call : dbms 내부에서 반입되는 call

<BR>

- Array Processing
  - 실무에서 One SQL을 작성하기 어려울 때, Array Processing 을 활용해 Call 부하를 줄일 수 있음
  - Array Size 크기만큼 묶어서 Call 하도록 조정

<BR>

- 인덱스, 제약 해제를 통한 대량 DML 튜닝
  - 배치 프로그램(동시 트랜잭션 없이 대량 데이터 적재)에서는
    - 인덱스 해제
    - 무결성 제약 조건 해제

<BR>

- 수정가능 조인 뷰
  - 조인 뷰 : FROM 절에 두 개 이상의 테이블을 가진 뷰
  - 수정가능 조인 뷰 : 입력, 수정, 삭제가 허용되는 조인 뷰
  - 키 보존 테이블 : 조인된 결과집합을 통해서도 중복 값 없이 Unique하게 식별이 가능한 테이블
